


| 对比维度      | SQL（关系型数据库）                                        | NoSQL（非关系型数据库）                                                                                                                                                  |
| :-------- | :------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **数据结构**  | 结构化数据，严格遵循预定义的表结构（字段、数据类型、长度固定），数据以行 / 列形式存储       | ![[Pasted image 20251124232011.png]]                                                                                                                            |
| **数据关联性** | 强关联型，通过「外键」明确表与表之间的关系（一对一、一对多、多对多），支持关联查询（JOIN）    | 弱关联 / 无关联型，不依赖外键建立关系；<br><br>**关系实现方式**：<br><br>1. 嵌入文档（如 JSON 文档中嵌套关联数据）；<br><br>2. 引用 ID（存储关联数据的唯一标识，需应用层手动关联）；<br><br>**缺点**：嵌入方式易导致数据大量重复（冗余），引用方式需额外处理关联逻辑 |
| **事务支持**  | 原生支持事务，严格遵循 ACID 特性（原子性、一致性、隔离性、持久性），适合数据一致性要求高的场景 | 早期 NoSQL 不支持事务或仅支持弱事务；<br><br>新型 NoSQL（如 MongoDB 4.0+、Cassandra）已支持部分 ACID 特性，但整体事务能力弱于 SQL 数据库（并非 “完全不满足 ACID”）                                                |
| **查询语法**  | 统一使用 SQL 标准语法（如 SELECT、INSERT、JOIN 等），语法规范统一       | 无统一 SQL 语法，各数据库有专属 API 或查询语言（如 MongoDB 的查询语法、Redis 的命令行语法）![[Pasted image 20251124231807.png]]                                                                  |
| **存储方式**  | 主要以磁盘存储为主，部分支持内存缓存加速                               | 存储方式多样：<br><br>1. 磁盘存储（如 MongoDB、Cassandra）；<br><br>==2. 内存存储（如 Redis，适合高并发场景）；==<br><br>==并非 “仅内存存储”==                                                         |
|           |                                                    | 低延迟：io 多路复用，基于内存                                                                                                                                                |
|           |                                                    | 多语言端：java/c/c++                                                                                                                                                 |
