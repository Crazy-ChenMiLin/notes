### 接口

1. **作用**：定义一组方法签名，规定了实现类必须要实现的功能，但不包含具体的实现逻辑。
2. **书写规范**：
    - 通常使用 `interface` 关键字声明接口。
    - 接口名一般以 `I` 开头（约定俗成，便于区分接口和类），首字母大写，采用驼峰命名法。
    - 在接口中声明方法签名，不写方法体。
    - 示例：

java

运行

```java
public interface IVoucherService {
    void createVoucher(Voucher voucher);
    Voucher getVoucherById(Long id);
}
```

### 实现类

1. **作用**：实现接口中定义的方法，提供具体的业务逻辑实现。
2. **书写规范**：
    - 使用 `class` 关键字声明类。
    - 类名一般是接口名去掉 `I` ，并加上 `Impl` 后缀（这是很常用的命名习惯，能清晰表明该类是某个接口的实现）。
    - 使用 `implements` 关键字指定要实现的接口，并实现接口中的所有方法。
    - 通常会加上 `@Service` 注解，将该类标记为 Spring 管理的服务类，让 Spring 容器能够自动扫描并创建其实例。
    - 示例：

java

运行

```java
@Service
public class VoucherServiceImpl implements IVoucherService {
    @Override
    public void createVoucher(Voucher voucher) {
        // 具体的创建凭证逻辑
    }

    @Override
    public Voucher getVoucherById(Long id) {
        // 具体的根据ID获取凭证逻辑
    }
}
```

### Controller 层

1. **作用**：负责接收前端请求，调用服务层的方法处理业务逻辑，然后返回相应的结果给前端。
2. **书写规范**：
    - 使用 `@RestController` 或 `@Controller` 注解声明类，`@RestController` 用于标注该类处理的请求返回的是 JSON 格式数据。
    - 定义处理请求的方法，使用 `@RequestMapping` 、`@GetMapping` 、`@PostMapping` 等注解来映射请求路径。
    - 通过依赖注入的方式，在 Controller 中声明接口类型的变量，Spring 会自动将对应的实现类实例注入进来。
    - 示例：

```java
@RestController
@RequestMapping("/voucher")
public class VoucherController {
    @Resource
    private IVoucherService voucherService;

    @GetMapping("/{id}")
    public Voucher getVoucherById(@PathVariable Long id) {
        return voucherService.getVoucherById(id);
    }
}
```
