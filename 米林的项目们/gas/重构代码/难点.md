# 1) 什么叫“出帧”
相机工作时，会以固定频率产生一张张图像（可以理解成视频的每一帧）。
相机每产生一张图像 = 出一帧
比如你代码里有设置帧率（我在 Grab_Callback 里看到有 AcquisitionFrameRate 相关设置），那它可能每秒出 5 帧、10 帧等。
出帧的结果不是“自动显示在你电脑上”，而是：相机通过驱动/SDK把这一帧的原始图像数据交给电脑程序。

# 2) 什么叫“回调”
回调（callback）就是：你把一个“函数/对象”交给 SDK，SDK 在合适的时机会反过来调用你提供的这个函数。
对比一下两种风格：
### A. 你主动去“拉取”图像（轮询/pull）
你程序写个循环，不断调用 SDK 的 getFrame() 之类的接口去拿：
while(true){
  frame = camera.getFrame();
}


### B. SDK 主动“推送”图像给你（回调/push）
你告诉 SDK：“相机有新图时，请调用我这个函数”。
这就是你现在的模式。
