当前链路基本是下面这样：
A. 入口 1：打开摄像头（WebSocket 触发）
前端连：/websocket/identify/{cameraId} → MvsWebSocket. OnOpen

OnOpen 做的事（代码事实）：
1. 如果该 cameraId 已经有旧连接/旧 grab 线程：先 closeMVS ()，清缓存、清裁剪参数、通知数据 websocket 清空

2. New Grab_Callback (...)（注意：直接 new，不是 Spring 注入）
3. 启动线程执行：grabCallback. OpenCamera (cameraId)

4. 把 Session 和 Grab_Callback 存在静态 Map 里

	GrabCallbackMap = 🎥 摄像头遥控器（控制硬件）
	按开机键（openCamera）
	按录制键（开始采集）
	按关机键（closeMVS）
	
	SessionMap = 📡 无线传输器（前后端通信）
	接收摄像头拍摄的画面
	通过 WiFi 传输给手机 APP
	让用户能在手机上实时看到画面


B. 摄像头帧回调（Grab_Callback 内部注册的 callback）
Grab_Callback 内会调用 SDK：MV_CC_RegisterImageCallBack (...)
回调里通常会做：拿到图像 → 编码/转换 →（可选裁剪）→（可选推理）→ 推送



C. 输出 1：推送视频帧（WebSocket）
MvsWebSocket.SendFrames (cameraId, bytes, width, height) 直接把二进制发给前端



D. 输出 2：推送识别数据（WebSocket）
另一个 websocket：/websocket/identify/data/{taskId} → GasDataWebSocket
GasDataWebSocket.SendData (taskId, List<String> data) 会把识别结果发给前端（异步文本）



E. 入口 2：枚举摄像头数量（HTTP Controller）
ImageController.GetCameraCount () 直接调用 SDK 的 MV_CC_EnumDevices (...)
也就是说：Controller 知道相机 SDK（这是典型耦合）
