2. @RequestBody 的工作原理是：
 Spring MVC 使用 HttpMessageConverter（特别是 Jackson 库）将请求体中的 JSON 数据转换为 Java 对象
 这个过程中，Spring 会直接通过反射机制创建对象实例，而不是从 Spring 容器中获取 Bean
创建的对象实例用于承载请求数据，然后传递给控制器方法

Spring MVC 框架看到 @RequestBody 注解后，会自动创建 LoginFormDTO 对象实例
将请求体中的 JSON 数据填充到该对象的字段中
将这个填充好的对象赋值给 loginForm 变量
因此您可以直接调用 loginForm.GetCode () 方法

3.Spring 框架在启动时自动调用的**！
![[Pasted image 20251128213858.png]]

整个流程是：

1. Spring 启动时，会扫描到你标注了 `@Configuration` 的 `MvcConfig` 类；
2. 发现 `MvcConfig` 实现了 `WebMvcConfigurer` 接口（这是 Spring 提供的「MVC 配置扩展接口」）；
3. Spring 会主动创建一个 `InterceptorRegistry` 类型的对象（这个对象是 Spring 内部管理的「拦截器注册器」，用来存储所有开发者注册的拦截器）；
4. Spring 调用你重写的 `addInterceptors` 方法，并把它创建好的 `InterceptorRegistry` 对象，通过「方法参数」的形式传给你（也就是你代码里的 `registry` 变量）；
