# Java 反射八股文

## 什么是反射？
反射（Reflection）是 Java 提供的一种强大的功能，使程序可以在运行时动态地加载类、获取类的信息、调用方法或操作字段。

## 反射的优缺点
### 优点：
1. **动态性**：在运行时加载类和调用方法，增强程序的灵活性。
2. **框架基础**：许多 Java 框架（如 Spring）大量依赖反射机制。
3. **可操作隐藏信息**：可以突破类的封装操作私有字段和方法（需小心使用）。

### 缺点：
1. **性能开销**：与直接调用相比，反射的性能较低。
2. **安全约束**：反射可能会破坏面向对象封装，操作私有成员。
3. **复杂性**：代码阅读性下降，容易导致 bug。

## 反射的核心类
Java 的反射主要通过以下几个核心类提供：
1. `Class` 类
2. `Method` 类
3. `Field` 类
4. `Constructor` 类

```java
Class<?> clazz = Class.forName("com.example.MyClass");
```

## 常见操作
### 加载类：
```java
Class<?> clazz = Class.forName("com.example.MyClass");
```
或
```java
Class<?> clazz = MyClass.class;
```

### 获取构造函数并实例化对象：
```java
Constructor<?> constructor = clazz.getConstructor(String.class);
Object obj = constructor.newInstance("参数");
```

### 操作字段：
```java
Field field = clazz.getDeclaredField("name");
field.setAccessible(true); // 允许操作私有字段
field.set(obj, "新值");
```

### 调用方法：
```java
Method method = clazz.getDeclaredMethod("methodName", 参数类型.class);
method.setAccessible(true);
Object returnValue = method.invoke(obj, 参数);
```

## 反射的应用场景
1. **框架开发**：如 Spring 和 Hibernate 的底层实现依赖反射。
2. **动态代理**：Java 动态代理机制使用反射来生成动态类。
3. **工具类**：如 JSON/XML 序列化与反序列化。
4. **注解解析**：运行时获取注解信息。

## 注意事项
- 尽量减少反射的直接使用，避免性能损耗。
- 权限控制：使用反射访问私有方法和字段需要注意安全与法律合规。
- 强烈依赖反射的代码可能难以维护，可用设计模式适当替代。
