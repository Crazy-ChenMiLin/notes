# 字节流与位流
### 1 字节（Byte）= 8 位（bit）：ASCII 编码（基础编码）

	规则：1 个字符 = 1 个字节（8 位）（实际仅用 7 位存储有效数据，最高位为 0）。

举例：字符 `'a'`、`'5'`、`'!'` 在 ASCII 编码下，都只占用 1 个字节（8 位）。

| 对比维度   | Word 文档读取（字节流）         | 霍夫曼压缩（位流）                   |
| ------ | ---------------------- | --------------------------- |
| 核心目标   | 正确解析已有标准格式，获取文档内容      | 最小化存储空间，实现极致压缩比             |
| 数据形态   | 完整字节序列（已字节对齐，无零散位）     | 大量零散位编码（5 位、3 位等，非字节对齐）     |
| 流的选择依据 | 符合计算机字节存储规则，有成熟库支持解析   | 突破字节边界，消除填充浪费，最大化空间利用率      |
| 核心操作   | 调用 POI 库，按字节读取、解析预定义格式 | 手动实现位级操作（移位、位与、位或），拼接 / 拆分位 |
| 通俗类比   | 拆快递：打开现成的标准包装盒，取出内部物品  | 打包快递：将零散小物件紧凑拼接，凑够箱子再封装     |
