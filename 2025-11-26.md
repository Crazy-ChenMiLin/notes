# Redis 使用笔记：Jedis 与 Spring Redis（Spring Boot 整合）

## 一、Jedis 使用步骤（原生 Redis 客户端）

Jedis 是 Redis 官方推荐的 Java 原生客户端，轻量直接，需手动管理连接，适合简单场景或需精细控制 Redis 连接的需求。

### 1. 前置准备

- 确保 Redis 服务已启动（本地 / 服务器），且端口（默认 6379）、密码（若配置）可访问。
- 项目中添加 Jedis 依赖（Maven/Gradle）。

### 2. 核心步骤（含代码示例）

#### 步骤 1：添加 Jedis 依赖（Maven）

xml

```xml
<!-- pom.xml 中添加依赖 -->
<dependency>
    <groupId>redis.clients</groupId>
    <artifactId>jedis</artifactId>
    <version>4.4.6</version> <!-- 推荐使用稳定版，需兼容 Redis 服务版本 -->
</dependency>
```

#### 步骤 2：创建 Jedis 实例（连接 Redis）

通过 `Jedis` 类指定 Redis 主机、端口、密码（若有），建立连接。

java

运行

```java
import redis.clients.jedis.Jedis;

public class JedisDemo {
    public static void main(String[] args) {
        // 1. 建立 Redis 连接（参数：主机IP、端口、超时时间、密码）
        Jedis jedis = new Jedis(
            "localhost", // Redis 主机（本地为 localhost，服务器填 IP）
            6379,        // 默认端口 6379
            3000         // 连接超时时间（毫秒）
        );
        
        // 2. 若 Redis 配置了密码，需认证（无密码则跳过）
        jedis.auth("your-redis-password"); // 替换为你的 Redis 密码
        
        // 3. 测试连接是否成功（返回 PONG 表示连接正常）
        String pong = jedis.ping();
        System.out.println("Redis 连接状态：" + pong); // 输出 PONG 则成功
    }
}
```

#### 步骤 3：操作 Redis 数据（常用命令）

Jedis 方法名与 Redis 原生命令一致，直接调用即可（如 `set`、`get`、`hset` 等）。

java

运行

```java
// 示例 1：操作字符串（String）
jedis.set("name", "张三"); // 存值：key=name，value=张三
String name = jedis.get("name"); // 取值：返回 "张三"
System.out.println("获取 name：" + name);

// 示例 2：操作哈希（Hash）
jedis.hset("user:1", "id", "1");    // 存哈希：key=user:1，field=id，value=1
jedis.hset("user:1", "age", "20");
String userId = jedis.hget("user:1", "id"); // 取哈希字段：返回 "1"
System.out.println("获取 user:1 的 id：" + userId);

// 示例 3：设置过期时间（key 30 秒后过期）
jedis.setex("code", 30, "123456"); // key=code，value=123456，过期时间 30 秒
```

#### 步骤 4：关闭连接（关键！避免资源泄露）

Jedis 实例是线程不安全的，使用后必须关闭连接。

java

运行

```java
// 操作完成后关闭连接
if (jedis != null) {
    jedis.close();
}
```

### 3. 进阶：使用 Jedis 连接池（推荐生产环境）

频繁创建 / 关闭 Jedis 实例会消耗资源，生产环境建议用连接池管理连接（线程安全，复用连接）。

java

运行

```java
import redis.clients.jedis.Jedis;
import redis.clients.jedis.JedisPool;
import redis.clients.jedis.JedisPoolConfig;

public class JedisPoolDemo {
    // 1. 配置连接池参数
    private static final JedisPoolConfig poolConfig = new JedisPoolConfig();
    // 2. 初始化连接池（静态代码块，只初始化一次）
    private static final JedisPool jedisPool;

    static {
        // 连接池配置：最大空闲连接、最大总连接、超时时间等
        poolConfig.setMaxIdle(10); // 最大空闲连接数
        poolConfig.setMaxTotal(50); // 最大总连接数
        poolConfig.setMaxWaitMillis(3000); // 获取连接的最大等待时间（毫秒）
        
        // 初始化连接池
        jedisPool = new JedisPool(
            poolConfig,
            "localhost",
            6379,
            3000,
            "your-redis-password"
        );
    }

    // 3. 从连接池获取 Jedis 实例
    public static Jedis getJedis() {
        return jedisPool.getResource();
    }

    // 4. 使用示例
    public static void main(String[] args) {
        try (Jedis jedis = getJedis()) { // try-with-resources 自动关闭连接
            jedis.set("gender", "男");
            System.out.println("获取 gender：" + jedis.get("gender"));
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

### 4. 注意事项

- Jedis 实例**线程不安全**，不可多线程共享一个实例，需通过连接池获取。
- 连接超时、密码错误、Redis 服务未启动会抛出 `JedisConnectionException`，需捕获异常。
- 依赖版本需与 Redis 服务版本兼容（如 Redis 6+ 推荐 Jedis 4.x+）。

## 二、Spring Redis 使用步骤（Spring Boot 整合，推荐）

Spring Redis 是 Spring 对 Redis 的封装（核心是 `RedisTemplate`/`StringRedisTemplate`），自动管理连接池，支持依赖注入，整合 Spring Boot 后配置简单，适合 Spring 生态项目。

### 1. 前置准备

- Spring Boot 项目（版本 2.x+ 或 3.x+ 均可）。
- Redis 服务已启动，可访问。

### 2. 核心步骤（含 Yaml 配置 + 依赖注入）

#### 步骤 1：添加 Spring Redis 依赖（Maven）

Spring Boot 提供 `spring-boot-starter-data-redis` Starter，自动引入核心依赖（Jedis 或 Lettuce 客户端，默认 Lettuce）。

xml

```xml
<!-- pom.xml 中添加依赖 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
    <!-- 若需强制使用 Jedis 而非默认的 Lettuce，添加以下排除依赖 + Jedis 依赖 -->
    <exclusions>
        <exclusion>
            <groupId>io.lettuce</groupId>
            <artifactId>lettuce-core</artifactId>
        </exclusion>
    </exclusions>
</dependency>
<!-- 强制使用 Jedis 客户端（可选，默认 Lettuce 无需此步） -->
<dependency>
    <groupId>redis.clients</groupId>
    <artifactId>jedis</artifactId>
</dependency>
```

#### 步骤 2：Yaml 配置 Redis（application.yml）

在 `src/main/resources/application.yml` 中配置 Redis 连接信息、连接池、序列化方式（关键！避免乱码）。

yaml

```yaml
spring:
  redis:
    # 1. Redis 基础连接配置
    host: localhost # Redis 主机（服务器填 IP）
    port: 6379      # 默认端口
    password: your-redis-password #  Redis 密码（无则注释或留空）
    database: 0     # 操作的 Redis 数据库（默认 0 号库，0-15 可选）
    timeout: 3000ms # 连接超时时间
    
    # 2. 连接池配置（Spring Boot 2.x+ 统一用 lettuce 或 jedis 节点配置）
    lettuce: # 若使用 Jedis，将 lettuce 改为 jedis 即可，配置参数一致
      pool:
        max-active: 8    # 最大连接数
        max-idle: 8      # 最大空闲连接数
        min-idle: 2      # 最小空闲连接数
        max-wait: -1ms   # 最大等待时间（-1 表示无限制）

# 3. 自定义 RedisTemplate 序列化配置（可选，但推荐，避免默认序列化乱码）
# （若不配置，默认用 JdkSerializationRedisSerializer，存储二进制数据，可视化乱码）
```

#### 步骤 3：配置 RedisTemplate（可选，解决序列化乱码）

默认 `RedisTemplate` 用 JDK 序列化，存储数据为二进制，推荐自定义配置为 JSON 序列化（Jackson），方便可视化和跨语言解析。

java

运行

```java
import com.fasterxml.jackson.annotation.JsonAutoDetect;
import com.fasterxml.jackson.annotation.PropertyAccessor;
import com.fasterxml.jackson.databind.ObjectMapper;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.redis.connection.RedisConnectionFactory;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.serializer.Jackson2JsonRedisSerializer;
import org.springframework.data.redis.serializer.StringRedisSerializer;

@Configuration
public class RedisConfig {

    // 自定义 RedisTemplate<String, Object>（Key 为 String，Value 为任意对象）
    @Bean
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory factory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        // 1. 关联 Redis 连接工厂
        template.setConnectionFactory(factory);
        
        // 2. 配置 JSON 序列化器（Value 序列化）
        Jackson2JsonRedisSerializer<Object> jacksonSerializer = new Jackson2JsonRedisSerializer<>(Object.class);
        ObjectMapper om = new ObjectMapper();
        // 允许访问对象的所有字段（包括 private）
        om.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        // 启用 Jackson 的默认类型（存储时带类名，反序列化时能正确转换类型）
        om.activateDefaultTyping(LaissezFaireSubTypeValidator.instance, ObjectMapper.DefaultTyping.NON_FINAL);
        jacksonSerializer.setObjectMapper(om);
        
        // 3. 配置 String 序列化器（Key 序列化）
        StringRedisSerializer stringSerializer = new StringRedisSerializer();
        
        // 4. 给 Key、Value 分别设置序列化器
        template.setKeySerializer(stringSerializer);       // Key 用 String 序列化
        template.setValueSerializer(jacksonSerializer);    // Value 用 JSON 序列化
        template.setHashKeySerializer(stringSerializer);   // Hash Key 用 String 序列化
        template.setHashValueSerializer(jacksonSerializer); // Hash Value 用 JSON 序列化
        
        // 5. 初始化模板
        template.afterPropertiesSet();
        return template;
    }
}
```

#### 步骤 4：依赖注入，使用 RedisTemplate/StringRedisTemplate

Spring Boot 自动注入 `RedisTemplate` 和 `StringRedisTemplate`（后者默认 String 序列化，适合 Key/Value 都是字符串的场景），通过 `opsForXXX()` 操作不同类型数据。

##### 示例 1：使用 StringRedisTemplate（字符串场景）

java

运行

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.stereotype.Service;

@Service
public class RedisStringService {

    // 依赖注入 StringRedisTemplate（默认 String 序列化，无需额外配置）
    @Autowired
    private StringRedisTemplate stringRedisTemplate;

    // 存值
    public void setString(String key, String value) {
        stringRedisTemplate.opsForValue().set(key, value);
    }

    // 取值
    public String getString(String key) {
        return stringRedisTemplate.opsForValue().get(key);
    }

    // 存值并设置过期时间（单位：秒）
    public void setStringWithExpire(String key, String value, long seconds) {
        stringRedisTemplate.opsForValue().set(key, value, seconds);
    }
}
```

##### 示例 2：使用自定义 RedisTemplate（对象场景）

java

运行

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Service;

@Service
public class RedisObjectService {

    // 依赖注入自定义的 RedisTemplate<String, Object>
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    // 存储对象（自动序列化为 JSON）
    public void setUser(String key, User user) {
        redisTemplate.opsForValue().set(key, user);
    }

    // 获取对象（自动反序列化为 User 类型）
    public User getUser(String key) {
        return (User) redisTemplate.opsForValue().get(key);
    }

    // 内部测试用 User 类
    static class User {
        private Integer id;
        private String name;
        private Integer age;
        // 必须有默认构造函数（Jackson 反序列化需要）
        public User() {}
        // getter/setter/toString 省略
    }
}
```

##### 示例 3：操作哈希（Hash）

java

运行

```java
@Service
public class RedisHashService {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    // 存储哈希数据
    public void setHash(String key, String hashKey, Object value) {
        redisTemplate.opsForHash().put(key, hashKey, value);
    }

    // 获取哈希字段值
    public Object getHash(String key, String hashKey) {
        return redisTemplate.opsForHash().get(key, hashKey);
    }
}
```

### 3. 关键说明

- **`StringRedisTemplate` vs `RedisTemplate`**：
    - `StringRedisTemplate`：Key/Value 均为 String 类型，默认 `StringRedisSerializer`，无乱码，适合字符串场景。
    - `RedisTemplate`：泛型类（需指定 `<K, V>`），默认 JDK 序列化，需自定义 JSON 序列化（如步骤 3），适合存储对象。
- **依赖注入**：Spring Boot 自动配置 `RedisConnectionFactory` 和 `RedisTemplate`，直接用 `@Autowired` 注入即可。
- **序列化**：必须配置序列化方式，否则存储对象时会出现乱码（二进制数据），推荐 Jackson 序列化。

### 4. 常见问题

- 注入 `RedisTemplate` 无提示：给 `RedisTemplate` 指定泛型（如 `RedisTemplate<String, Object>`），IDEA 即可识别类型。
- 连接超时：检查 Redis 主机、端口是否正确，服务器防火墙是否开放 6379 端口，Redis 配置 `bind` 是否允许当前 IP 访问。
- 序列化失败：确保对象有默认构造函数，字段有 getter/setter，Jackson 依赖已正确引入（`spring-boot-starter-data-redis` 已包含）。

## 三、两者对比与选择

|特性|Jedis（原生客户端）|Spring Redis（Spring Boot 整合）|
|---|---|---|
|连接管理|手动管理（需用连接池）|自动管理（内置连接池）|
|配置复杂度|较高（需手动配置连接池、序列化）|较低（Yaml 简单配置，自动注入）|
|整合 Spring 生态|不支持依赖注入，需手动整合|原生支持，无缝融入 Spring 项目|
|序列化|需手动处理（如 JSON 转字符串）|内置多种序列化器，支持自动 JSON 序列化|
|适用场景|简单项目、非 Spring 项目、需精细控制连接|Spring Boot/Spring 项目、生产环境、存储对象|

**推荐选择**：Spring Boot 项目优先用 `Spring Redis`（`RedisTemplate`/`StringRedisTemplate`），配置简单、无需手动管理连接，开发效率高；非 Spring 项目或需精细控制 Redis 连接的场景，用 Jedis + 连接池。
## RedisTemplate 是什么？

[RedisTemplate](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/test/java/org/example/springredis/SpringRedisApplicationTests.java#L14-L14) 是 Spring Data Redis 提供的一个核心类，它是 Spring 对 Redis 操作的封装模板类。你可以把它理解为操作 Redis 数据库的"万能钥匙"。

## 为什么要在代码中使用 RedisTemplate？

### 1. **统一的操作接口**
[RedisTemplate](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/test/java/org/example/springredis/SpringRedisApplicationTests.java#L14-L14) 提供了一套统一的 API 来操作 Redis，无论你是操作字符串、哈希、列表还是集合，都可以通过它来完成。

### 2. **自动配置和连接管理**
当你在 pom. Xml 中添加了 `spring-boot-starter-data-redis` 依赖并在 [application.yaml](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/main/resources/application.yaml) 中配置了 Redis 连接信息后，Spring Boot 会自动创建 [RedisTemplate](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/test/java/org/example/springredis/SpringRedisApplicationTests.java#L14-L14) Bean 并配置好连接。

在你的测试代码中：

```java
@Autowired
private RedisTemplate redisTemplate;
```


这一行通过 Spring 的依赖注入机制，自动获取了配置好的 [RedisTemplate](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/test/java/org/example/springredis/SpringRedisApplicationTests.java#L14-L14) 实例。

### 3. **不同类型数据的操作方法**

在你的测试方法中：
```java
redisTemplate.opsForValue().set("name", "chenmilin");
Object name = redisTemplate.opsForValue().get("name");
```


这里使用了 `opsForValue()` 方法，它是专门用来操作字符串类型的 Redis 数据。RedisTemplate 为每种 Redis 数据类型都提供了对应的操作方法：

- `opsForValue()` - 操作字符串 (String)
- `opsForHash()` - 操作哈希 (Hash)
- `opsForList()` - 操作列表 (List)
- `opsForSet()` - 操作集合 (Set)
- `opsForZSet()` - 操作有序集合 (ZSet)

### 4. **连接池和性能优化**
通过 [RedisTemplate](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/test/java/org/example/springredis/SpringRedisApplicationTests.java#L14-L14)，Spring Boot 可以自动管理 Redis 连接池，避免每次都创建新连接的开销，提高性能。

## 示例说明

在你的测试代码中：
```java
@Test
void testString() {
    // 使用 RedisTemplate 操作 Redis 数据库
    redisTemplate.opsForValue().set("name", "chenmilin");  // 设置键值对
    Object name = redisTemplate.opsForValue().get("name"); // 获取值
    System.out.println("name是"+name);                     // 输出结果
}
```


这段代码实际上完成了以下操作：
1. 通过 [RedisTemplate](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/test/java/org/example/springredis/SpringRedisApplicationTests.java#L14-L14) 连接到你在 [application.yaml](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/main/resources/application.yaml) 中配置的 Redis 服务器（192.168.100.128:6379）
2. 使用字符串操作方法设置了一个键值对：`name` -> `chenmilin`
3. 然后又通过相同的键获取值并打印出来

这就像使用 JDBC 模板操作数据库一样，[RedisTemplate](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/test/java/org/example/springredis/SpringRedisApplicationTests.java#L14-L14) 让你不需要关心底层连接细节，只需专注于数据操作。

总的来说，[RedisTemplate](file://C:/Users/26487/Desktop/MiLin%E7%9A%84%E9%A1%B9%E7%9B%AE%E4%BB%AC/Jedis-demo/spring-redis/src/test/java/org/example/springredis/SpringRedisApplicationTests.java#L14-L14) 是 Spring Boot 中操作 Redis 的标准方式，它简化了 Redis 的使用，提供了统一的编程接口，并且自动处理了连接管理和性能优化。
