### 核心知识点整理

## 一、接口参数校验

- 手机号格式校验：通过`RegexUtils.isPhoneInvalid(phone)`判断手机号格式是否合法，格式错误时返回`Result.fail("手机号格式错误")`；格式合法则返回`Result.ok("发送成功")`（验证码发送场景）。
	发送验证码，返回
	Return Result. Ok ("发送成功");

```java
if (RegexUtils.isPhoneInvalid(phone)) { // 关键行
    return Result.fail("手机号格式错误"); // 返回“格式错”提示
}
```
## 二、Java 强类型特性与类型转换

- `HttpSession.getAttribute(String name)`返回值为`Object`类型（可存储 / 获取任意类型对象），若需赋值给`String`类型变量，需**强制类型转换**（Java 强类型语言要求类型匹配）。

## 三、String 字符串比较规则

- 字符串比较必须使用 `equals` 方法（而非 `==`，`==` 比较地址，`equals` 比较内容）

#### 1. 基本类型（int、long、double 等）用 `==`

```java
int goodsId1 = 1001;
int goodsId2 = 1001;
System.out.println(goodsId1 == goodsId2); // true（直接比较数值）

double price1 = 299.9;
double price2 = 299.9;
System.out.println(price1 == price2); // true（基本类型值比较）
```

#### 2. 包装类 / 数值型
（Integer、Long、BigDecimal 等）
用 `equals()` 或 `compareTo()`

``` java
    Integer a = 100;
    Integer b = new Integer(100);
    System.out.println(a == b); // false（地址不同）
    System.out.println(a.equals(b)); // true（值相同）
    ```



## 四、HttpSession 核心机制

1. 创建与关联：客户端首次访问服务器时，Servlet 容器（如 Tomcat）自动创建`HttpSession`对象；服务器通过 Cookie 或 URL 重写维护客户端与 Session 的关联。
2. 生命周期：由 Servlet 容器自动管理，用于存储用户会话数据。

## 五、基于 Session 的权限控制

- 登录态校验逻辑：在需要登录的接口中，检查 Session 中是否存在 "user" 对象：
    
    - 存在：用户已登录，允许访问接口；
    - 不存在：需重新登录。
    

## 六、Redis 序列化方式适配性

| 序列化方式  | 适合 Key 吗？ | 适合 Value 吗？ | 核心原因                                   |
| ------ | --------- | ----------- | -------------------------------------- |
| String | 超适合       | 仅适合纯文本      | Key 需简洁无冗余，String 满足；Value 无法存储复杂数据    |
| JSON   | 超不适合      | 超适合         | Key 会产生冗余字符，影响查询性能；Value 可存储任意结构化数据并还原 |

## 七、Java 编译与运行（IDE 视角）

| 阶段          | 触发方式                             | 核心动作                                | 典型异常 / 提示                            |
| ----------- | -------------------------------- | ----------------------------------- | ------------------------------------ |
| 编译（Compile） | IDE 自动触发（保存代码）/ 手动 Build Project | 将`.java`源码编译为 JVM 可执行的`.class`字节码文件 | `Cannot resolve symbol '类名'`（符号解析失败） |
| 运行（Run）     | IDE 点击 Run / 执行测试类               | JVM 读取`.class`文件并执行逻辑               | `NoClassDefFoundError`（类未找到）         |

## 八、多态（编译时 / 运行时类型）

- 编译时类型：IDE 跳转方法声明时，基于接口 / 父类类型（如`ValueOperations`接口）；
- 运行时类型：程序执行 /debug 跟进时，基于实现类类型（如`DefaultValueOperations`实现类）。
