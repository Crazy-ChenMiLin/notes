### Synchronized:
同时具备"悲观锁的设计思想"和“互斥锁的功能特性”

| 对比维度     | synchronized             | ReentrantLock                 |
| -------- | ------------------------ | ----------------------------- |
| **实现方式** | Java 关键字（JVM 层面实现）       | Java 类（API 层面实现）              |
| **锁释放**  | 自动释放（方法 / 代码块结束）         | 手动释放（必须在 finally 中 unlock ()） |
| **锁类型**  | 非公平锁（默认）                 | 可指定公平 / 非公平（构造参数）             |
| **可中断性** | 不可中断（线程阻塞后无法唤醒）          | 可中断（lockInterruptibly () 方法）  |
| **超时获取** | 不支持（阻塞时一直等）              | 支持（tryLock (long time)，超时放弃）  |
| **条件变量** | 支持（通过 wait ()/notify ()） | 支持（通过 Condition，更灵活）          |
| **适用场景** | 简单场景（如普通同步）              | 复杂场景（如公平锁、超时控制）               |


### Synchronized: 互斥锁的原因：
	1. 同一时间，已有一个线程能持有锁：无论锁的是this/类对象（xx. Class）/自定义对象（new object）
	2. 未强到锁的进程会被阻塞
