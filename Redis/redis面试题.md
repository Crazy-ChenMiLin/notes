### ~~4. 什么情况 Redis 会丢失数据~~

1. ~~未开持久化：Redis 进程崩溃，内存数据直接丢失~~
2. ~~RDB 间隔内故障：丢失两次快照间的数据~~
3. ~~AOF 同步策略：用`everysec/no`时，故障会丢部分命令~~
4. ~~主从同步延迟：主节点写数据后未同步给从节点就宕机~~
5. ~~集群脑裂：主节点脱离集群后接收的写请求，后续会丢失~~

### 6. Redis 使用场景

1. 缓存：热点数据缓存（减轻 DB 压力）
2. 分布式锁：解决资源竞争（如商品超卖）
3. 计数器：实时统计点赞数 / 阅读量
4. 排行榜：ZSet 实现热搜 / 游戏排名
5. 消息队列：List 实现简单队列
6. 限流：incr + 过期时间实现接口限流

### 7. 分布式锁除了超卖还能干什么

1. 资源互斥：多服务抢稀缺资源（如演唱会门票）
2. 防止重复：避免定时任务多实例重复执行、接口重复提交
3. 事务控制：保证跨服务操作的原子性
4. 资源同步：多服务更新同一配置 / 缓存时，避免冲突
