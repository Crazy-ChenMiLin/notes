为什么要有两个？

深拷贝和对象属性拷贝

不能简单说 “final 完全不能修改”，**final 的 “不可修改” 是分场景的**—— 要看它修饰的是类、方法，还是变量（还要分变量是基本类型 / 引用类型）：

### 1. 修饰「类」：不能修改 “继承关系”

final 修饰的类，**不能被其他类继承**，相当于这个类的 “结构和行为” 不能被子类修改（比如 String 类是 final，不能写`class MyString extends String {}`）。

### 2. 修饰「方法」：不能修改 “方法实现”

final 修饰的方法，**不能被子类重写**，也就是子类不能修改这个方法的逻辑（比如 Object 的 getClass () 是 final，子类不能重新写这个方法的内容）。

### 3. 修饰「变量」：分两种情况

这是最容易混淆的点，要区分**基本类型变量**和**引用类型变量**：

- **修饰基本类型变量**：值完全不能修改（比如`final int num=10;`，num 不能再赋值成其他数字）。
- **修饰引用类型变量**：**引用本身不能修改（不能指向其他对象），但对象的内容可以修改**（比如 `final StringBuilder sb = new StringBuilder("Hello");`，不能让 sb 换成其他 StringBuilder 对象，但可以用 `sb.append("World")` 修改 sb 里的字符串内容）。


`static` 的 “共享” 是指 **“所有地方访问的是同一个资源”**，但 `final` 的 “不可修改” 是指**“这个资源的内容不能被改”**—— 二者是 “共享同一个不可变的东西”，完全不冲突。
```java
// Math类里的定义：公共、类共享、不可修改的圆周率常量
public static final double PI = 3.141592653589793;
```
