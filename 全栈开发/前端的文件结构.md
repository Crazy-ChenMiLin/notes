### 一、核心目录 & 文件解析

| 目录 / 文件          | 作用说明                                                    | 小白视角理解                                          |
| ---------------- | ------------------------------------------------------- | ----------------------------------------------- |
| `node_modules`   | 存放项目依赖的第三方包（如 Vue、Vite、axios 等），由 npm 自动管理，**不需要手动修改**。 | 相当于 “工具库”，项目运行需要的 “零件” 都存在这里。                   |
| `src`（核心开发目录）    | 所有业务代码都写在这里，是开发的重点区域。                                   | 你写的所有前端功能（页面、组件、接口等）都在这个文件夹里。                   |
| ├─ `api`         | 封装后端接口请求函数（如`login.js`负责登录接口的调用）。                       | 专门放 “和后端通信” 的代码，比如登录、获取数据等操作。                   |
| ├─ `components`  | 存放可复用的组件（如`Login.vue`是登录页面的组件）。                         | 把页面拆成一个个 “小模块”，比如按钮、弹窗、登录框，方便重复使用。              |
| ├─ `router`      | 配置页面路由（`index.js`定义 “访问哪个地址显示哪个页面”）。                    | 相当于 “导航地图”，决定用户点击链接后跳转到哪个页面。                    |
| ├─ `utils`       | 存放工具函数（如`request.js`封装了 Axios，统一处理接口请求、Token 携带等）。      | 存放 “通用功能” 的代码，比如发送网络请求、时间格式化等，哪里需要哪里用。          |
| ├─ `views`       | 存放页面级的组件（如`Home.vue`是首页）。                               | 这里的每个文件对应一个完整的 “页面”，比如首页、个人中心页。                 |
| ├─ `App.vue`     | 根组件，是所有页面的 “容器”，可写全局布局（如导航栏、页脚）。                        | 整个应用的 “大框架”，所有页面都会嵌套在这个组件里。                     |
| ├─ `main.js`     | 项目入口文件，负责初始化 Vue 应用、挂载根组件、引入路由等。                        | 项目的 “启动器”，相当于 “从这里开始执行前端代码”。                    |
| `index.html`     | Vite 的入口 HTML 文件，Vue 应用会挂载到这个文件的指定节点上。                  | 网页的 “骨架”，所有前端功能最终都会嵌入到这个 HTML 中展示。              |
| `package.json`   | 管理项目依赖和脚本（如`npm run dev`启动项目的命令就配置在这里）。                 | 相当于 “项目说明书”，记录了项目需要哪些依赖、怎么启动 / 打包。              |
| `vite.config.js` | Vite 的配置文件（如配置路径别名、开发服务器、插件等）。                          | 给 Vite “下命令” 的文件，比如设置 “@” 别名指向`src`目录，就是在这里配置的。 |

### 二、项目运行流程（从启动到页面展示）

1. 执行 `npm run dev`（在终端中运行），Vite 会读取`vite.config.js`的配置，启动开发服务器。
2. 浏览器访问 `http://localhost:5173`（默认端口），加载`index.html`。
3. `index.html` 引入 `main.js`，`main.js` 初始化 Vue 实例，挂载 `App.vue`。
4. `App.vue` 根据 `router/index.js` 的路由配置，渲染对应的页面（如`Login.vue`或`Home.vue`）。
5. 当用户操作页面（如点击登录按钮），`Login.vue` 会调用 `api/login.js` 中的接口函数，这些函数通过 `utils/request.js` 发送请求到后端，完成交互。

### 三、小白注意点

- 冗余文件：`vite.config.js` 和 `vite.config.mjs` 保留一个即可（建议留`vite.config.js`，并确保是 ESM 语法）。
- 依赖安装：如果新增了第三方包（如`axios`），需要执行 `npm install 包名` 来安装，否则会报 “模块找不到” 的错误。
- 开发流程：修改`src`内的文件后，Vite 会自动热更新（页面实时刷新），无需手动重启。

这样梳理后，你可以先从`main.js`和`App.vue`入手，逐步理解 “页面是如何渲染、接口是如何调用” 的，慢慢就能掌握项目的整体逻辑啦～
