## 1. 直接本地打开（纯静态页面）

- **操作**：找到项目里的`index.html`，双击直接用浏览器打开。
- **适用场景**：只有 HTML/CSS/JS 的简单静态页面（无 AJAX 请求、无依赖打包）。
- **缺点**：如果页面有跨域请求（比如调接口），浏览器会拦截报错。

## 2. 用 npm 启动开发服务器（现代前端项目）

- **操作**：项目根目录执行 `npm run dev`（或 `yarn dev`、`pnpm dev`）
-—— ==依赖`package.json`里配置的脚本（比如 Vite、Webpack、React/Vue 项目的开发命令==
- **适用场景**：用构建工具（Vite、Webpack）开发的项目，自带热更新、跨域代理等功能，是前端开发的主流方式。


## 3. 线上部署后访问

**操作**：
- 把打包后的 `dist` 目录部署到服务器（Nginx/Apache），
- 或静态托管平台（GitHub Pages、Netlify、Vercel），通过域名访问。
**适用场景**：项目上线后，给用户访问的正式环境。

# 总结：
- npm 开发服务器：只在**写代码时用**（开发阶段），帮你实时看效果。
- Nginx：只在**代码写完、上线后用**（生产阶段），帮用户访问最终页面。
