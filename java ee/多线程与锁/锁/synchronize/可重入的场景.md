## 场景 1：同一类中「同步方法嵌套调用同步方法」（最常见）

### 场景背景

实际开发中，一个业务方法（加了同步锁）往往需要调用同类的另一个同步方法，比如：

- 银行账户类：`transfer()`（转账，同步方法）需要调用`deductBalance()`（扣余额，同步方法）和`addBalance()`（加余额，同步方法）；
- 订单类：`confirmOrder()`（确认订单，同步方法）需要调用`lockStock()`（锁库存，同步方法）。

### 可重入的体现

同一线程执行外层同步方法时，调用内层同步方法（锁对象相同），不会卡死，能正常执行。

``` java
### 代码示例（银行转账）
public class BankAccount {
    // 实例级共享资源：账户余额
    private int balance;
    // 锁对象（实例方法的锁是this，同步代码块也锁this，保证锁对象一致）
    private final Object lock = new Object();

    // 外层同步方法：转账（锁this）
    public synchronized void transfer(BankAccount target, int money) {
        // 1. 扣减当前账户余额（调用内层同步方法）
        this.deductBalance(money);
        // 2. 增加目标账户余额
        target.addBalance(money);
        System.out.println("转账完成，当前账户余额：" + this.balance);
    }

    // 内层同步方法：扣余额（锁this，和外层是同一把锁）
    public synchronized void deductBalance(int money) {
        if (this.balance < money) {
            throw new RuntimeException("余额不足");
        }
        this.balance -= money;
    }

    // 内层同步方法：加余额（锁this）
    public synchronized void addBalance(int money) {
        this.balance += money;
    }

    // 测试
    public static void main(String[] args) {
        BankAccount a = new BankAccount();
        a.balance = 1000;
        BankAccount b = new BankAccount();
        b.balance = 500;

        // 单线程调用transfer，触发嵌套同步方法调用
        a.transfer(b, 200);
    }
}
```
