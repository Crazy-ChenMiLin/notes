数据库中的 char
定义: 在数据库中，char 是一种固定长度的字符数据类型，用于存储字符数据。
长度: 通常可以指定长度，如 char (10)，表示最多存储 10 个字符。
存储: 固定长度，如果存储的字符少于定义的长度，数据库会用空格填充剩余位置。
编码: 通常使用字符集编码（如 UTF-8, GBK 等）来存储字符。
Java 中的 char
定义: Java 中的 char 是一种基本数据类型，用于表示单个 16 位 Unicode 字符。
长度: 固定为 16 位（2 字节）。
存储: 存储的是 Unicode 字符，范围从 \u 0000 到 \uffff（0 到 65535）。
编码: 使用 Unicode 编码。



## 题目 1：创建学生表
**题目描述**：创建一个学生表，要求：
- 学号为主键且自增长
- 姓名必须非空
- 年龄必须≥18（使用检查约束）
- 手机号必须唯一（使用唯一约束）

```sql
CREATE TABLE 学生表 (
    学号 INT PRIMARY KEY AUTO_INCREMENT,
    姓名 VARCHAR(20) NOT NULL,
    年龄 TINYINT CHECK (年龄 >= 18),
    手机号 CHAR(11) UNIQUE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 题目 2：创建课程表
**题目描述**：创建一个课程表，要求：
- 课程 ID 为主键且自增长
- 课程名称必须非空
- 学分必须在 1-6 之间（使用检查约束）

```sql
CREATE TABLE 课程表 (
    课程ID INT PRIMARY KEY AUTO_INCREMENT,
    课程名称 VARCHAR(50) NOT NULL,
    学分 INT CHECK (学分 BETWEEN 1 AND 6)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 题目 3：创建成绩表（含外键）
**题目描述**：创建成绩表，要求：
- 成绩 ID 为主键且自增长
- 学号为外键，引用学生表的学号
- 课程 ID 为外键，引用课程表的课程 ID
- 分数必须在 0-100 之间（使用检查约束）

```sql
CREATE TABLE 成绩表 (
    成绩ID INT PRIMARY KEY AUTO_INCREMENT,
    学号 INT,
    课程ID INT,
    分数 INT CHECK (分数 BETWEEN 0 AND 100),
    CONSTRAINT fk_学生 FOREIGN KEY (学号) REFERENCES 学生表(学号),
    CONSTRAINT fk_课程 FOREIGN KEY (课程ID) REFERENCES 课程表(课程ID)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 题目 9：分组查询与 HAVING
**题目描述**：查询每个部门的平均年龄，只显示平均年龄大于 30 的部门。

```sql
SELECT 部门, AVG(年龄) AS 平均年龄 
FROM 员工表 
GROUP BY 部门 
HAVING 平均年龄 > 30;
```

## 题目 12：创建订单表（含 DECIMAL 类型）
**题目描述**：创建一个订单表，要求：
- 订单 ID 为主键且自增长
- 用户 ID 为外键，引用用户表
- 订单金额必须精确到分（使用 DECIMAL (10,2)）
- 订单日期使用 DATE 类型

```sql
CREATE TABLE 订单表 (
    订单ID INT PRIMARY KEY AUTO_INCREMENT,
    用户ID INT,
    订单金额 DECIMAL(10, 2),
    订单日期 DATE,
    CONSTRAINT fk_用户 FOREIGN KEY (用户ID) REFERENCES 用户表(用户ID)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 题目 15：分页查询
**题目描述**：查询所有订单的订单 ID、用户 ID 和订单金额，按订单金额降序排列，并只显示前 5 条记录。
