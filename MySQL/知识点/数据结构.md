# 提问
#### 问题 3：选择 MySQL 数据类型时的核心原则是什么？请举例说明不同场景下的类型选择。

**答案**：核心原则：① 匹配数据范围，选择 “能覆盖数据范围且存储空间最小” 的类型（节省资源）；② 确保精度需求，精确数值（如货币）用 DECIMAL，非精确数值（如身高）用 FLOAT/DOUBLE；③ 考虑存储效率，短文本用 CHAR/VARCHAR，长文本用 TEXT（但避免直接存储超大文本，建议存文件路径）。举例：① 年龄（0-120）→ TINYINT unsigned；② 商品价格（精确到分）→ DECIMAL (10,2)；③ 用户名（1-20 字符）→ VARCHAR (20)；④ 文章内容（超 65535 字符）→ LONGTEXT；⑤ 注册时间→ TIMESTAMP（自动记录）。
